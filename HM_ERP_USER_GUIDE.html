1. Introduction

1.1 About HM ERP

HM ERP is a comprehensive Enterprise Resource Planning system designed specifically for manufacturing and retail businesses. It integrates all aspects of your business operations from production to sales, providing real-time visibility and control.

Key Features

  • Production Management: Track orders through multiple production stages
  • Inventory Control: Manage raw materials and finished goods
  • Point of Sale (POS): Retail sales and customer management
  • Financial Tracking: Comprehensive accounting and reporting
  • Employee Management: Attendance, production tracking, and performance
  • Multi-Branch Support: Manage multiple locations from one system
  • Real-time Reporting: Instant insights into business performance

1.2 System Architecture

The system is built on modern cloud technology, ensuring:

  • 24/7 availability from anywhere
  • Automatic backups and data security
  • Scalability as your business grows
  • Mobile-responsive design for on-the-go access

5. Production Module

5.1 Production Stages

The system tracks products through the following stages:

Stage Description Typical Duration
PENDING Order received, awaiting approval 1-2 days
CUTTING Raw material cutting and preparation 1-2 days
SHAPING Forming and shaping components 1-2 days
BENDING Bending metal sheets as required 1 day
WELDING_INNER Internal welding work 2-3 days
WELDING_OUTER External welding and assembly 2-3 days
GRINDING Surface finishing and smoothing 1-2 days
FINISHING Final touches and quality check 1 day
PAINTING Paint application and drying 2-3 days
PLYWOOD_FITTING Plywood installation (if applicable) 1 day
COMPLETED Ready for delivery -

5.2 Order Management Workflow

9.2 Daily Operations Workflow

9.2.1 Operator Daily Routine

  1. Morning:
    • Check in via Attendance system
    • Review assigned orders
    • Check machine status
  2. During Shift:
    • Start production on assigned orders
    • Record production progress
    • Log material usage
    • Report any issues or wastage
    • Take breaks (system tracks break time)
  3. End of Shift:
    • Complete pending production entries
    • Check out via Attendance
    • Hand over to next shift (if applicable)

9.2.2 Store Manager Daily Routine

  1. Opening:
    • Check inventory levels
    • Review pending stock transfers
    • Prepare POS system
  2. During Day:
    • Process customer sales via POS
    • Receive stock transfers
    • Handle customer queries
    • Monitor inventory
  3. Closing:
    • Reconcile cash/payments
    • Review daily sales
    • Update low stock items
    • Generate daily report

10. Troubleshooting

10.1 Common Issues & Solutions

Issue: Cannot Delete Employee

Solution: This issue has been fixed. Employees can now be deleted even if they have related records (attendance, production logs, etc.). The system automatically handles cascading deletions.

Issue: Customer Creation Fails

Solution: This issue has been fixed. Multiple customers can now have the same phone number or no phone number. The unique constraint has been removed.

Issue: Bills Page Shows Dashboard

Solution: This issue has been fixed. ADMIN users can now access the Bills page correctly.

Issue: Login Problems

Solutions:

  • Verify email and password are correct
  • Check if account is active
  • Clear browser cache and cookies
  • Try a different browser
  • Contact admin for password reset

Issue: Page Not Loading

Solutions:

  • Check internet connection
  • Refresh the page (F5 or Ctrl+R)
  • Clear browser cache
  • Check if you have permission to access that page

Issue: Data Not Saving

Solutions:

  • Check all required fields are filled
  • Verify data format (dates, numbers, etc.)
  • Check for error messages on screen
  • Try again after a few moments
  • Contact support if issue persists

10.2 Best Practices

System Usage Guidelines

  • Regular Backups: System automatically backs up data, but keep local records of critical information
  • Accurate Data Entry: Always double-check information before saving
  • Timely Updates: Record production, attendance, and sales in real-time
  • Security: Never share login credentials; log out when finished
  • Training: Ensure all users are properly trained on their modules
  • Communication: Report issues immediately to prevent workflow disruption
  • Documentation: Keep physical records as backup for critical transactions
